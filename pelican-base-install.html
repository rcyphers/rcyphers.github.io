Pelican Quickstart: Install Pelican on Ubuntu 16.04

Date Wed 13 September 2017 Tags HowTo / pelican / virtualenv

Verified with: Ubuntu 16.04, Python 2.7.12, Pelican v-3.7.1

A static website generator uses a markup language, such as Markdown or reStructuredText, and a templating engine such as Jinja, to produce HTML files. Static sites can be hosted easily on github pages, Amazon S3 and even dropbox. Static site generators offer a lot of control over a web site through templates and plugins without the complexity dealing with a hosting provider or Content Management System (CMS).

Static sites are also more secure because there are fewer moving parts in the deployed website.

I chose Pelican as a static site generator based on its support for Markdown, Pygments, a large number of themes and plugins and of course being hackable in Python.

Why Choose a Static Site Generator?

The typical CMS driven website works by building each page on-demand, fetching content from a database and running it through a template engine. This means each page is assembled from templates and content on each request to the server.

A Static Site Generator takes a different approach and generate all the pages of the website only when there’s actually changes to the site. This means there’s no moving parts in the deployed website. Caching gets much easier, performance goes up and static sites are far more secure.

Install Virtualenv and Virtualenvwrapper

Virtualenv is a tool for creating Python environments isolated from the global Python installation and other virtualenv environments. Virtualenv makes it easier to work on multiple projects without introducing conflicts in dependencies or breaking system-wide packages.

Virtualenvwrapper provides a set of commands which simplifies working with virtual environments. It also places all your virtual environments in one place.

sudo apt-get update
sudo apt-get upgrade

sudo apt-get install wget curl git unzip
sudo apt-get install python-pip
sudo -H pip install --upgrade pip
pip install --user virtualenv
pip install --user virtualenvwrapper

cd
echo '#' >> ~/.bashrc
echo '# virtualenvwrapper setup' >> ~/.bashrc
echo 'export WORKON_HOME=$HOME/.virtualenv' >> ~/.bashrc
echo 'export PROJECT_HOME=$HOME/virtenv-projects' >> ~/.bashrc
echo 'source $HOME/.local/bin/virtualenvwrapper.sh' >> ~/.bashrc

source ~/.bashrc
mkdir $PROJECT_HOME

Install Pelican

Create a virtual environment to isolate the pelican package and its dependencies.

The mkproject myproject command creates the virtual environment in the $WORKON_HOME, and a project directory in $PROJECT_HOME. The workon myproject command will automatically cd to the project directory.

mkproject pelican
workon pelican

Install pelican in the virtualenv. If you plan on using Markdown as a markup format, you’ll need to also install the Markdown library.

pip install pelican
pip install --upgrade pelican
pip install Markdown typogrify

Add support for blogging with Jupyter/IPython Notebooks using the pelican-ipynb plugin (pelican-ipynb).

pip install jupyter ipython nbconvert beautifulsoup4

It’s a good idea to “freeze” the current state of the environment packages for future reference. To do this, run:

pip freeze > requirements.txt

Install Pelican Themes and Plugins

Pelican comes with no plugins and two very basic themes, so you have to clone the needed ones to a local directory. Here I clone the whole official plugins and themes repositories, allowing me to experiment with different options.

Themes and plugins can be cloned directly into the themes and plugins subdirectories of the Pelican site-packages (e.g. $WORKON_HOME/pelican/lib/python2.7/site-packages/pelican/). Instead I clone them to my home directory, so will need to specify the location in the pelican configuration file later.

git clone --recursive https://github.com/getpelican/pelican-themes ~/pelican-themes

git clone --recursive https://github.com/getpelican/pelican-plugins ~/pelican-plugins

Test it out

At this point you’re ready to create a test blog. Run pelican-quickstart to start an sequence of interactive of questions that help with the initial configuration. For most of the questions, it’s okay to just hit Enterand accept the default value.

The ones you may want to modify iniitially include:

  • the title of the website
  • the author of the website
  • n for the URL prefix

  • the timezone

    pelican-quickstart

Customize the pelican config file (pelicanconf.py)

Settings are configured in the form of a Python module (a file). There is an example settings file available for reference.

ref: Pelican > Docs > Settings

Edit the file pelicanconf.py to specify a theme, activate plugins and tell pelican where to find plugins and themes:

THEME = '../../pelican-themes/pelican-bootstrap3'
JINJA_ENVIRONMENT = {'extensions': ['jinja2.ext.i18n']}
PLUGIN_PATHS = ["../../pelican-plugins/"]
MARKUP = ('md', 'ipynb')
PLUGINS = [ 'i18n_subsites', 'ipynb.markup']
